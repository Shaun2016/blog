负载均衡与反向代理

发表于 2020-01-25 | 分类于 website architecture

当一个服务器无法支撑用户请求时，需要扩容，从一台服务器扩展到很多台。然而，用户访问时是通过同一个域名，浏览器会查询DNS服务器获取对应的IP，然后通过此IP访问服务。反向代理就是将很多台服务器对外看起来是一台服务器，用反向代理服务器选择具体哪个服务器为用户提供服务

负载均衡算法

  • round-robin (default)：轮询
  • ip_hash：相同ip的请求发送到相同的服务器
  • hash key
    • 哈希算法
    • 一致性哈希算法
  • least_conn：将请求发送到最小活跃连接的上游服务器上

当fail_timeout内失败了max_fails次请求，认为该上游服务器不可用，移除该服务器，尝试下一台服务器

健康检查

  • TCP心跳检查

interval 每隔3秒检查一次，timeout 连接超时为失败一次，fall 失败3次后标记为不存活，rise 成功1次后标记为存活

  • HTTP 心跳检查

check_http_send: 检查时发送的HTTP请求内容；check_http_expect_alive: 当上游服务器返回匹配的响应状态码时，则认为上游服务器存活

域名上游服务器

社区版 Nginx 不支持域名对应 IP 的动态更新；商业版支持；proxy_pass http://aaa.com 是支持动态域名解析

备份上游服务器

将 8002 端口的服务器配置为备上游服务器，当所有主上游服务器都不存活时，请求会转发给备上游服务器

不可用上游服务器

Nginx Location 匹配规则

用于匹配请求字符串，用不同的模块去处理不同匹配的请求：location [=|~|~*|^~] uri {...}，其中uri是要去匹配的字符串，可以不包含正则，称为“标准uri”；包含正则的uri称为“正则uri”

标识符 描述 例子
= 精确匹配，用于标准uri location = /hello/ {} 对应 curl 192.168.0.1/hello/
~ 正则匹配，用于正则uri，区分大小写 location ~ \.(html) {} 对应 192.168.0.1/aaa.html
~* 正则匹配，用于正则uri，不区分大小写 location ~ \.(html) {} 对应 192.168.0.1/aaa.html
^~ 非正则匹配，匹配前缀，用于标准uri， location ^~ /hello/ {} 对应 192.168.0.1/index/...
/ 全部请求
